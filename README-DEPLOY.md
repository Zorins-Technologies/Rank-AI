# 🚀 Rank AI: Production Deployment Guide

This guide provides step-by-step instructions for deploying **Rank AI** to production using **Render** (Backend) and **Vercel** (Frontend).

---

## 🛠️ Phase 1: Backend Deployment (Render)

1. **Create Service**: Link your GitHub repo to Render and choose the `backend` directory as the build context.
2. **Settings**:
   - **Environment**: `Node`
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
3. **Environment Variables**:
   Add the following secrets to the Render "Environment" tab:

| Variable | Recommended Value | Description |
| :--- | :--- | :--- |
| `NODE_ENV` | `production` | Enables production hardening |
| `PORT` | `8000` | Render will override this automatically |
| `ALLOWED_ORIGINS` | `https://your-app.vercel.app` | Your frontend URL (comma-separated for multiple) |
| `GCP_PROJECT_ID` | `your-gcp-id` | Google Cloud Project ID |
| `GCS_BUCKET_NAME` | `your-bucket` | Google Cloud Storage Bucket |
| `GENAI_API_KEY` | `AI...` | Gemini 1.5 Flash API Key |
| `GOOGLE_CLOUD_CLIENT_EMAIL` | `account@...` | From your service account JSON |
| `GOOGLE_CLOUD_PRIVATE_KEY` | `-----BEGIN...` | From your service account JSON (Ensure no extra quotes) |
| `DB_USER` | `postgres` | Cloud SQL username |
| `DB_PASSWORD` | `your-db-password` | Cloud SQL password |
| `DB_NAME` | `your-db-name` | Cloud SQL database name |
| `INSTANCE_CONNECTION_NAME` | `project:region:instance` | Cloud SQL connection string |
| `USE_SECRET_MANAGER` | `false` | Set to `true` if using Secret Manager to fetch DB credentials |

## 🌐 Phase 2: Frontend Deployment (Vercel)

1. **Import Project**: Choose the `frontend` directory in the Vercel import screen.
2. **Framework**: Vercel will auto-detect **Next.js**.
3. **Environment Variables**:
   Add these to the Vercel dashboard:

| Variable | Value | Description |
| :--- | :--- | :--- |
| `NEXT_PUBLIC_API_URL` | `https://rank-ai-api.onrender.com/api` | **Must include /api** |
| `NEXT_PUBLIC_FIREBASE_API_KEY` | `AI...` | From Firebase Console |
| `NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN` | `...firebaseapp.com` | From Firebase Console |
| `NEXT_PUBLIC_FIREBASE_PROJECT_ID` | `your-id` | Matches Backend GCP ID |
| `NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET`| `...appspot.com` | From Firebase Console |
| `NEXT_PUBLIC_FIREBASE_APP_ID` | `1:123...` | From Firebase Console |

## ☁️ Phase 3: Serverless Backend (Google Cloud Run)

If you prefer a scalable, serverless container environment, follow these steps:

1. **Build the Container**:
   Run this from the `backend` directory:
   ```bash
   gcloud builds submit --tag gcr.io/your-project-id/rank-ai-backend
   ```
2. **Deploy to Cloud Run**:
   ```bash
   gcloud run deploy rank-ai-service \
     --image gcr.io/your-project-id/rank-ai-backend \
     --platform managed \
     --region us-central1 \
     --allow-unauthenticated \
     --add-cloudsql-instances=your-project:us-central1:your-instance
   ```
3. **Environment Variables**:
   You can set them in the Cloud Run Console or via the CLI:
   - `GCP_PROJECT_ID`, `GENAI_API_KEY`, `GCS_BUCKET_NAME`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`, `INSTANCE_CONNECTION_NAME`, `ALLOWED_ORIGINS`
   - If you're using Google Secret Manager instead of direct env vars, set `USE_SECRET_MANAGER=true` and ensure the secrets exist with the names from `backend/src/services/secret.service.js`.
   - For service account credentials, either supply a valid `GOOGLE_APPLICATION_CREDENTIALS_JSON`, or store `GOOGLE_CLOUD_PRIVATE_KEY` and `GOOGLE_CLOUD_CLIENT_EMAIL` as secrets. The backend now normalizes escaped newlines and removes extra surrounding quotes automatically.

## ✅ Deployment Checklist

> [!IMPORTANT]
> **CORS Sync**: If you add a custom domain later, remember to update the `ALLOWED_ORIGINS` on Render.

> [!WARNING]
> **Firebase Authorized Domains**: In the **Firebase Console > Authentication > Settings**, ensure you add your Vercel URL (e.g., `rank-ai.vercel.app`) to the **Authorized Domains** list, or Google Login will fail.

> [!TIP]
> **Trailing Slashes**: I have updated the code to automatically handle trailing slashes in your API URL, but it is best practice to provide it exactly as `https://your-api.com/api`.
